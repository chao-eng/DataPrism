# 📦 项目需求文档 (PRD)：rou-tools

## 一、 项目概述
**rou-tools** 是一款专为光谱/波长强度数据处理设计的桌面端工具。核心解决用户面对大量包含“波长-时间-强度”数据的 Excel 文件时，手工提取繁琐、突刺数据清洗困难的问题。该工具提供从**多文件批量提取 -> 智能算法清洗 -> 可视化人工审核 -> 归总导出**的完整闭环工作流。

## 二、 技术栈选型
该工具采用现代化轻量级桌面端架构：
* **桌面端框架**：Tauri (提供原生系统交互与高性能后端)
* **前端框架**：Vue 3 (基于 Vite 构建，保证极速冷启动与热更新)
* **UI 样式**：Vanilla CSS (原生 CSS，保持轻量级)
* **图标库**：Lucide Icons (简洁一致的矢量图标)
* **图表库**：ECharts 
* **本地数据库**：SQLite (用于持久化存储原始数据、清洗数据与任务记录)

## 三、 核心业务流程
1.  **启动/主页**：打开工具，进入任务概览页。
2.  **创建任务**：指定数据源（包含多个结构化 Excel 文件的文件夹） -> 输入/选择目标波长。
3.  **提取与落库**：按文件顺序，提取目标波长对应的整行强度数据，**原始数据**按文件名标记保存至 SQLite。
4.  **算法清洗**：对提取的原始数据应用【基于局部中位数的严苛 Z-Score 过滤算法】，剔除突刺点，**清洗后数据**保存至 SQLite。
5.  **可视化审核**：在界面生成指定文件名的波长强度散点图进行预览。
6.  **人工干预**：在散点图中发现异常点，可手动点击剔除，并同步更新数据库。
7.  **导出交付**：审核无误后，将所有处理好的数据汇总，导出为标准格式的 Excel 文件。

## 四、 功能模块详细需求

### 1. 首页与任务管理模块
* **任务列表展示**：展示历史执行过的所有任务。列表信息应包含：任务名称/时间、处理文件数量、目标波长、任务状态（处理中/待审核/已完成）。
* **任务回溯**：支持点击历史任务，直接进入该任务的“可视化审核与导出”界面。
* **新建任务入口**：提供显眼的“新建任务”按钮。

### 2. 任务创建模块
* **数据源选择**：调用系统文件选择器，让用户选择一个**文件夹**（程序需自动扫描并识别该文件夹下的所有 Excel 文件）。
* **波长配置**：提供输入框或下拉列表，供用户指定需要提取的目标波长值（对应原始文件第一列的某个值）。
* **执行触发**：点击“开始执行”后，进入数据处理流程，界面需提供进度条或加载提示。

### 3. 核心数据处理引擎 (后端/Tauri侧执行)
* **数据提取**：
    * 遍历读取选定文件夹内的每个 Excel 文件。
    * 定位第一列，匹配用户指定的“目标波长”。
    * 提取该波长所在行的后续所有列数据（随时间变化的强度值）。
    * **落库要求**：将这批数据作为【原始数据】存入 SQLite，并与“源文件名”建立强关联映射。
* **智能数据清洗**：
    * **算法核心**：严格执行基于“局部中位数”的 Z-Score 算法。
    * **目标**：免疫极端突刺干扰，精准识别并剔除极高/极低的异常散点，使散点云极度收束。
    * **落库要求**：清洗过滤后的序列作为【清洗后数据】存入 SQLite。

### 4. 数据可视化与人工审核模块
* **图表渲染**：
    * 为任务下的每一个文件提供预览视图。
    * **图表类型**：散点图 (Scatter Plot)。
    * **坐标系定义**：
        * **X 轴**：自然数序列（1, 2, 3, 4... 代表采样点先后顺序）。
        * **Y 轴**：波长强度值。
* **交互式人工校准**：
    * 支持用户在散点图上进行鼠标交互（缩放、平移）。
    * **手动剔除**：用户点击选定某个散列点后，可将其删除（或标记为无效）。
    * **数据同步**：人工剔除操作触发后，实时保存更新至 SQLite 数据库中的【清洗后数据】记录，图表实时重绘。

### 5. 结果导出模块
* **导出触发**：用户确认预览图表无误后，点击“导出 Excel”按钮。
* **导出格式标准**：
    * 第一行（表头）：各源文件的文件名（如 `10-1000W.xlsx`, `2A.xlsx` 等）。
    * 第二行：当前提取的目标波长值。
    * 第三行及以后：纵向依次排列该文件经过算法清洗 + 人工校准后的强度序列数据（空缺值留空处理）。

## 五、 数据结构设计建议 (SQLite)
1.  **Task 表**：记录任务基础信息（id, created_at, target_wavelength）。
2.  **FileRecord 表**：记录任务包含的文件清单（id, task_id, file_name）。
3.  **DataPoint 表**（核心大表）：
    * 字段：id, file_id, x_index, original_value, filtered_value, is_deleted (0/1标记算法或人工剔除)。
